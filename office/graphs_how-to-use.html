StenoGraphs help: How to use graphs Guide

A practical reference for choosing chart types, preparing data, and getting crisp, honest visuals.

Overview

StenoGraphs supports ten chart types: bar, column, line, area, pie, histogram, scatter, radar, bubble, and waterfall. This page shows when to use each, how to format data (JSON or CSV), and quick best practices.

Quick steps
  1. Pick a chart type that matches your question.
  2. Prepare data in JSON or CSV with the right columns.
  3. Set titles, axes, palette, and legend placement.
  4. Check readability (labels, grid, contrast), then export.
CSV vs JSON
  • CSV: fastest for tables. First row = headers; first column = labels (most charts).
  • JSON: precise control; supports scatter/bubble objects like {x,y,r}.

Scatter/Bubble CSV: use columns x,y,label or x,y,r,label.

Data formats

JSON pattern

{
  "labels": ["Jan","Feb","Mar"],
  "datasets": [
    { "label": "Revenue", "data": [12, 19, 13] },
    { "label": "Cost", "data": [8, 11, 9] }
  ]
}

Scatter/Bubble use arrays of points instead of numbers:

{
  "labels": [],
  "datasets": [
    { "label": "Set A", "data": [{ "x": 1, "y": 2 }, { "x": 2, "y": 3 }] }
  ]
}

Chart-by-chart guidance

Bar and column

Use for categorical comparisons; long labels (bar), time buckets (column).

  • Data: labels + 1–N series of numbers.
  • Tip: Use bar for horizontal (better for long labels).
  • Avoid: too many categories; group or filter to the top 8–12.
Labels,North,South

B,18,15

Line

Use for trends over ordered time; multiple series ok.

  • Smoothing: small tension (0.2–0.4) for gentle curves, keep truth to data.
  • YAxis: start at zero only if meaningful; otherwise use auto to keep detail.
{
  "labels": ["Jan","Feb","Mar","Apr"],
  "datasets": [
    {"label":"Revenue", "data":[12,19,13,25]},
    {"label":"Cost", "data":[8,11,9,12]}
  ]
}

Area

Use for cumulative feel, stacked contribution over time.

  • Stacking: enable “Stacked” for parts-of-trend (e.g., product lines).
  • Transparency: keep fills ~25–35% opacity to avoid occlusion.
{
  "labels": ["2019","2020","2021","2022","2023"],
  "datasets": [
    {"label":"iOS","data":[20,25,30,42,50]},
    {"label":"Android","data":[28,33,35,38,44]}
  ]
}

Pie

Use for single snapshot part-to-whole with ≤6 slices.

  • Labels: sort slices by size and avoid many tiny categories.
  • Alternative: use bar for precise comparisons.

Histogram

Use for distribution of continuous values.

  • Bins: try 10–20; adjust to reveal shape without overfitting noise.
  • Data: a single series of raw numbers; StenoGraphs bins for you.
{
  "labels": [],
  "datasets": [{ "label": "Measurements", "data": [1.2,1.5,1.7,1.8,2.2,2.6,2.8,3.0] }]
}

Scatter

Use for relationships and correlations.

  • Axes: both numeric; add units in axis titles.
  • Series: group points by label to compare clusters.
{
  "labels": [],
  "datasets": [
    {"label":"Set 1","data":[{"x":1,"y":1},{"x":2,"y":1.2},{"x":3,"y":2.5}]},
    {"label":"Set 2","data":[{"x":1,"y":0.8},{"x":2,"y":1.7},{"x":3,"y":1.9}]}
  ]
}

Bubble

Use for 3 variables: x, y, and magnitude (r).

  • Scaling: keep bubble sizes within a readable range (e.g., 6–24 px radii).
  • Legend: clarify what radius encodes.
{
  "labels": [],
  "datasets": [
    {"label":"Markets","data":[{"x":10,"y":20,"r":12},{"x":15,"y":12,"r":18}]}
  ]
}

Radar

Use for comparing profiles across several axes for a few items.

  • Limit: ≤6–8 axes; more becomes cluttered.
  • Normalize: consistent scales (e.g., 0–100) across axes.
{
  "labels": ["Speed","Reliability","Usability","Features","Security","Support"],
  "datasets": [
    {"label":"Product A","data":[65,59,90,81,56,55]},
    {"label":"Product B","data":[28,48,40,19,96,27]}
  ]
}

Waterfall

Use for stepwise contributions to a final total.

  • Start: include a starting value labeled “Start” if applicable.
  • Total: last bar is auto-calculated “Total.”
{
  "labels": ["Start","Sales","COGS","Marketing","R&D","Other","Net"],
  "datasets": [{ "label":"Cashflow", "data":[100,60,-30,-10,-8,-2] }]
}

Style and readability

  • Palette: pick contrasty colors; avoid misleading hues for diverging data.
  • Grid lines: keep light; disable if they fight with the data.
  • Legend: position where labels are closest to data (top/right is typical).
  • Labels: enable “Show values” sparingly; prioritize clean axes and tooltips.
  • Axes: title with units (e.g., “Revenue (USD)”).
  • Scale: don’t truncate axes to exaggerate differences (unless justified).
  • Annotations: keep it minimal—one callout beats a label soup.

Exporting and sharing

  • PNG export: use a light chart background for print; dark for slides.
  • Aspect ratio: 16:9 for slides, 4:3 for reports, Square for dashboards.
  • Config copy: copy chart config to reproduce a figure later with new data.

Troubleshooting

  • Empty chart: check CSV headers and first column labels; verify numeric cells.
  • Weird scatter/bubble: confirm x and y are numbers, not strings.
  • Histogram is flat: increase bins or verify your data isn’t all identical.
  • Pie labels overlap: reduce slices; group small categories into “Other.”
